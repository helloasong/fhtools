## 目标
- 数据预览显示列名且更易读
- 数据导入过程异步化并显示进度，消除卡顿
- 合并 EDA & Binning 为一个紧凑的界面，缩小图表、提升美观度

## 界面调整
- 创建 `CombinedView`（替代 EDA 与 Binning 两页）：
  - 左侧：变量列表 + 搜索/筛选
  - 右侧上方（紧凑卡片式）：变量基础统计（类型、样本数、缺失率、唯一值、最值、均值、STD、分位数），每项使用较小字号与列栅格布局
  - 右侧中部：
    - 小尺寸分布图（直方/KDE + 类别条形），固定高度（如 200px），减少边距（`setContentsMargins(0,0,0,0)`）
    - 小尺寸分箱图（柱状=样本数、折线=Bad Rate），固定高度（如 220px），折线与柱状透明度优化
  - 右侧下方：分箱配置（算法、箱数、缺失策略、运行按钮、保存），布局紧凑，控件间距减小
- 美化：
  - 使用 `qt-material` 主题的密集风格（密集控件间距），统一字体、色彩与图例
  - 所有 PlotWidget 设置较小字体、紧凑边距、隐藏不必要图例

## 数据预览列名与表格优化
- `QTableView + DataFrameModel`：
  - 确保 `headerData` 返回列名（已支持），强制显示水平表头
  - 启用 `horizontalHeader().setStretchLastSection(True)` 与 `setSectionResizeMode`，提升列名可见性
  - 顶部显示选中文件名与形状（rows×cols）

## 异步导入与进度
- 导入流程改为后台线程（`Worker(QThread)`）：
  - 阶段进度：
    1. 文件读取（CSV: pandas chunk/headers；Excel: 读取与解析；Parquet：快速读取）
    2. 基础 EDA 计算（统计与类型识别）
    3. 控制器状态更新与视图刷新
  - 进度条：在 Import 区域显示不定进度或分阶段进度；完成后弹提示并切换到 `CombinedView`
  - UI 响应：导入期间禁用相关按钮，结束后恢复

## 控制器与服务调整
- `ProjectController.create_new_project` 改为仅触发 Worker；Worker 内调用 `repository.load_data` 与 `calculate_eda`，完成后发信号：`data_loaded(df)`、`project_updated(state)`
- 变量选择变化时同步刷新统计、分布、分箱图与指标表
- 运行分箱时保持当前紧凑布局与图表联动（悬停 Tooltip 继续保留）

## 图表压缩与布局
- 统一设置：
  - `PlotWidget.setFixedHeight(200~220)`，紧凑边距 `plotItem.layout.setContentsMargins(0,0,0,0)`
  - 坐标轴字体减小；折线与柱状透明度优化
  - 取消冗余网格/图例，必要时用简化图例
