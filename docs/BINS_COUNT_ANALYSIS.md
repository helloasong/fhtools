# 分箱数量分析与改进方案

**问题**: 当前默认 min_bins=2，保底也是2分箱，但2分箱的业务价值有限  
**目标**: 在保证单调性的前提下，尽量保持至少3箱

---

## 一、业务价值分析

| 箱数 | 业务价值 | 适用场景 | 问题 |
|-----|---------|---------|------|
| 2箱 | ⭐⭐ 低 | 快速筛选 | 区分度不足，难以做精细策略 |
| 3箱 | ⭐⭐⭐ 中 | 高中低分层 | 基本可用，但粒度较粗 |
| 4-5箱 | ⭐⭐⭐⭐ 高 | 标准评分卡 | 平衡了区分度和稳定性 |
| 6-8箱 | ⭐⭐⭐⭐⭐ 最高 | 精细化运营 | 区分度高，但可能过拟合 |

**结论**: 业务上通常期望 **至少3箱**，4-5箱为最佳。

---

## 二、当前问题分析

### 2.1 代码现状

```python
DEFAULT_MIN_BINS = 2  # 默认最小箱数

def _fallback_bins(self, x):
    """保底2分箱"""
    median = x.median()
    return [-np.inf, median, np.inf]
```

### 2.2 可能产生2箱的情况

1. **数据本身无法分裂** - 决策树只能产生1个切点
2. **强制合并过度** - 为了单调性，合并到只剩2箱
3. **IV损失过大** - 尝试更少箱数以满足iv_tolerance

---

## 三、改进方案

### 方案1: 提高默认 min_bins 到 3（推荐）

**改动**:
```python
DEFAULT_MIN_BINS = 3  # 从2改为3
```

**影响**:
- ✅ 避免产生2箱
- ⚠️ 可能增加IV损失（被迫保留更多箱数）
- ⚠️ 可能降低单调达成率（无法合并到2箱保底）

### 方案2: 改进保底策略为3分箱

**改动**:
```python
def _fallback_bins(self, x):
    """保底3分箱 - 33%和66%分位数"""
    p33 = x.quantile(0.33)
    p67 = x.quantile(0.67)
    return [-np.inf, p33, p67, np.inf]
```

**影响**:
- ✅ 保底至少3箱
- ⚠️ 保底箱数增加，但IV可能不如2箱

### 方案3: 添加箱数惩罚机制

在 auto 模式下，优先保留更多箱数：

```python
# 在 merge 和 pava 都可行时，选择箱数更多的
if merge_bins >= min_bins and pava_bins >= min_bins:
    if merge_bins > pava_bins:
        choose_merge()
    elif pava_bins > merge_bins:
        choose_pava()
    else:
        # 箱数相同，选IV损失小的
        choose_by_iv_loss()
```

---

## 四、决策

**采用方案1+方案2组合**:
1. 默认 min_bins 从2提高到3
2. 保底策略从2分箱改为3分箱
3. 保留用户自定义 min_bins=2 的灵活性

**理由**:
- 业务上3箱是最低可接受标准
- 如果数据确实只能分2箱，用户可显式设置 min_bins=2
- 与竞品对比（等频/等宽都是固定箱数），我们需要保持竞争力
