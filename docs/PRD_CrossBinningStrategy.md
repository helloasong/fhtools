# FHBinningTool - 组合策略分析需求文档 (PRD)

> **版本**: v1.0  
> **日期**: 2026-05-17  
> **作者**: AI Assistant  
> **状态**: 需求评审中  
> **功能定位**: 基于已有单变量分箱结果，通过笛卡尔积交叉组合挖掘多变量联合策略规则

---

## 1. 背景与目标

### 1.1 背景

FHBinningTool 当前的核心能力是对**单个变量**进行最优分箱，输出 WOE、IV、坏账率等指标。但在实际风控策略开发中，**单变量的极端区间往往不足以捕捉高风险客群**——真正的高风险人群通常表现为多个维度的联合异常，例如：

- "年龄 < 30 **且** 收入 < 5000 **且** 查询次数 > 5" 的客群坏账率可能高达 35%
- 单独看"年龄 < 30"坏账率只有 8%，单独看"收入 < 5000"坏账率只有 10%
- 但三个条件**同时满足**时，风险呈指数级放大

### 1.2 目标

为 FHBinningTool 增加**组合策略分析**模块，支持：

1. **基于已有分箱结果的交叉组合**：无需重新分箱，直接利用已确认的单变量分箱边界
2. **策略规则自动筛选**：自动找出高区分度的多变量联合区间
3. **可视化决策矩阵**：二维场景下支持热力图直观展示风险分布
4. **规则导出**：支持导出为 IF-THEN 策略规则，可直接用于风控决策引擎

### 1.3 与单变量分箱的关系

| 维度 | 单变量分箱 | 组合策略分析 |
|------|-----------|-------------|
| **分析对象** | 单个变量的区间分布 | 多个变量联合区间的分布 |
| **最优性** | 单变量最优切分 | 基于单变量切分的笛卡尔积组合（非联合最优） |
| **输出形式** | WOE、IV、单调性 | 策略规则（IF-AND-THEN） |
| **应用场景** | 评分卡变量筛选、WOE编码 | **风控策略规则挖掘**、特征交叉衍生 |
| **使用时机** | 变量级分析 | **批量分箱完成后**的策略挖掘阶段 |

> **关键设计原则**：本功能**不重新计算分箱边界**，严格复用 `ProjectState.binning_configs` 中已保存的切分点，确保策略规则与评分卡分箱结果完全一致。

## 2. 术语定义

| 术语 | 定义 |
|------|------|
| **组合策略 (Cross Strategy)** | 两个及以上变量的箱区间通过 AND 条件组合形成的联合规则 |
| **决策矩阵 (Decision Matrix)** | 二维变量交叉组合的 bad_rate 热力图 |
| **笛卡尔积组合 (Cartesian Combination)** | 各变量箱数的乘积，即所有可能的联合区间总数 |
| **Lift** | 组合区间坏账率 / 整体坏账率，衡量风险放大倍数 |
| **规则命中率** | 满足某组合条件的样本占总样本的比例 |

## 3. 功能需求

### 3.1 核心功能

#### FR-001: 基于已有分箱结果的交叉组合
- **描述**: 用户选择 2~N 个已分箱变量，系统基于各变量的分箱切分点做笛卡尔积组合
- **细节**:
  - 变量来源：必须从 `ProjectState.binning_configs` 中已存在分箱结果的变量中选择
  - 未分箱变量不允许选择，UI 上以灰色禁用状态展示
  - 箱标签映射：使用 `pd.cut(x, bins=cfg.splits)` 将原始值映射到箱标签
  - 缺失值处理：遵循各变量分箱时的 `missing_strategy` 配置（单独成箱 / 忽略 / 归并）
  - 组合数上限：当笛卡尔积组合数 > 5,000 时，提示用户"组合数过多，建议减少变量数或合并部分变量的箱"

#### FR-002: 策略规则筛选
- **描述**: 对笛卡尔积组合的结果应用筛选条件，只保留有业务价值的联合区间
- **筛选参数**:

| 参数 | 默认值 | 范围 | 说明 |
|------|--------|------|------|
| `min_sample_rate` | 1% | 0.5% ~ 10% | 组合区间最小样本占比 |
| `bad_rate_mode` | relative | relative / absolute | 坏账率阈值模式 |
| `bad_rate_high_multiplier` | 2.0 | 1.0 ~ 5.0 | **高于**整体坏账率的倍数（找高风险） |
| `bad_rate_low_multiplier` | 0.5 | 0.1 ~ 1.0 | **低于**整体坏账率的倍数（找优质客群） |
| `min_lift` | 1.5 | 1.0 ~ 5.0 | 最小 Lift 值 |
| `sort_by` | bad_rate_desc | bad_rate_desc / lift_desc / sample_desc | 结果排序方式 |

- **筛选逻辑**:
  ```
  保留条件 = (样本占比 >= min_sample_rate)
           AND (
               (坏账率 >= 整体坏账率 × bad_rate_high_multiplier)
               OR (坏账率 <= 整体坏账率 × bad_rate_low_multiplier)
           )
           AND (|Lift| >= min_lift)
  ```
- **风险等级自动标注**:
  - `bad_rate >= 整体 × 3.0` → 🔴 极高风险
  - `bad_rate >= 整体 × 2.0` → 🟠 高风险
  - `bad_rate <= 整体 × 0.5` → 🟢 优质客群
  - 其他 → ⚪ 正常

#### FR-003: 二维决策矩阵可视化
- **描述**: 当用户恰好选择 **2 个变量** 时，展示二维热力图
- **可视化规格**:
  - X轴：变量A的各箱（按箱中值排序）
  - Y轴：变量B的各箱（按箱中值排序）
  - 颜色：bad_rate 的高低，使用红-白-绿 diverging colormap
    - 红色 = 高于整体坏账率
    - 白色 = 接近整体坏账率
    - 绿色 = 低于整体坏账率
  - 单元格标注：显示样本数（小字）和 bad_rate（大字）
  - 交互：鼠标悬停显示详细统计（样本数、好坏样本数、WOE、Lift）
- **多变量场景（>2个）**: 隐藏热力图，只展示规则列表（高维无法在二维平面展示）

#### FR-004: 规则列表展示
- **描述**: 以表格形式展示所有满足筛选条件的组合策略
- **表格列定义**:

| 列名 | 说明 | 示例 |
|------|------|------|
| `规则编号` | 自增序号 | Rule-001 |
| `风险等级` | 图标+文字 | 🔴 极高风险 |
| `组合条件` | 可读的 IF 条件 | `age∈[20,30) ∧ income∈[0,5000)` |
| `样本数` | 该组合区间样本量 | 1,250 |
| `占比` | 占总样本比例 | 6.25% |
| `坏样本数` | bad count | 438 |
| `坏账率` | bad / total | 35.04% |
| `整体坏账率` | 基准对比 | 12.00% |
| `Lift` | 风险放大倍数 | 2.92 |
| `WOE` | 证据权重 | 0.8876 |
| `操作` | 复制/导出/标记 | [复制规则] |

- **排序**: 默认按坏账率降序，用户可点击表头切换排序
- **分页**: 当结果 > 100 条时，自动分页展示（每页 50 条）

#### FR-005: 规则导出
- **描述**: 将选中的策略规则导出为多种格式
- **导出格式**:
  1. **Excel**: 规则列表 + 每个规则的详细统计表
  2. **文本规则**: 纯文本 IF-THEN 格式
     ```
     IF age >= 20 AND age < 30 AND income >= 0 AND income < 5000 THEN 高风险 (bad_rate=35.04%, lift=2.92)
     ```
  3. **JSON**: 结构化规则，可直接被决策引擎消费
     ```json
     {
       "rule_id": "Rule-001",
       "conditions": [
         {"variable": "age", "operator": "in", "range": [20, 30]},
         {"variable": "income", "operator": "in", "range": [0, 5000]}
       ],
       "bad_rate": 0.3504,
       "lift": 2.92,
       "sample_count": 1250,
       "sample_rate": 0.0625
     }
     ```

### 3.2 交互功能

#### FR-006: 变量选择与预览
- **描述**: 在分析前提供已选变量的信息预览
- **预览内容**:
  - 每个已选变量的箱数
  - 各变量的分箱方法（智能分箱/等频/卡方等）
  - 笛卡尔积总组合数实时计算
  - 如果某变量箱数过多（>10箱），提示"箱数过多可能导致组合爆炸"

#### FR-007: 规则收藏与标记
- **描述**: 用户可对感兴趣的规则打星收藏
- **细节**:
  - 收藏状态仅保存在当前会话（不持久化到 .fht 文件）
  - 收藏的规则置顶显示
  - 支持"只看收藏"筛选

#### FR-008: 反向规则挖掘
- **描述**: 除找高风险组合外，同时找出显著低风险的优质客群组合
- **细节**:
  - 通过 `bad_rate_low_multiplier` 参数控制
  - 优质客群规则在表格中以 🟢 标识
  - 可用于"白名单策略"或"营销高响应客群"挖掘

## 4. 非功能需求

### 4.1 性能需求

| 需求编号 | 场景 | 要求 |
|---------|------|------|
| NFR-001 | 2变量交叉（各5箱） | < 1秒（10万样本） |
| NFR-002 | 3变量交叉（各5箱） | < 3秒（10万样本） |
| NFR-003 | 4变量交叉（各5箱） | < 10秒（10万样本），需提前警告组合数 |
| NFR-004 | 组合数 > 5,000 | 拒绝执行，提示减少变量或合并箱 |

### 4.2 用户体验需求

| 需求编号 | 需求 | 说明 |
|---------|------|------|
| NFR-005 | 入口可见性 | CombinedView 左侧必须有明显的入口按钮，已分箱变量≥2时才可点击 |
| NFR-006 | 空状态提示 | 没有已分箱变量时，显示"请先对至少2个变量进行分箱" |
| NFR-007 | 筛选结果为空 | 当筛选条件过于严格导致无结果时，提示"当前筛选条件下未找到有效组合，建议放宽阈值" |
| NFR-008 | 计算进度 | 组合数 > 1,000 时显示进度条，避免用户以为卡死 |

### 4.3 数据一致性需求

| 需求编号 | 需求 | 说明 |
|---------|------|------|
| NFR-009 | 分箱边界一致性 | 必须使用 `binning_configs[feature].splits`，不允许重新计算切分 |
| NFR-010 | 过滤规则继承 | 组合分析时应应用各变量的有效过滤规则（全局/自定义） |
| NFR-011 | 目标变量一致性 | 使用 `ProjectState.target_col`，与单变量分箱保持一致 |

## 6. 边界情况与异常处理

| 场景 | 处理策略 |
|------|---------|
| 用户选择了未分箱变量 | 变量列表中未分箱变量灰色禁用，不可勾选 |
| 笛卡尔积组合数 > 5,000 | 分析按钮禁用，提示"组合数 xxxx 过多，建议减少变量数或先合并部分变量的箱" |
| 筛选后结果为空 | 显示空状态插图 + 文字"未找到满足条件的组合策略，建议放宽筛选阈值" |
| 某组合区间样本数为0 | 自动过滤，不显示在结果中 |
| 过滤后数据为空 | 提示"应用过滤规则后数据为空，请检查各变量的过滤设置" |
| 目标变量未设置 | 按钮禁用，提示"未设置目标变量，无法进行策略分析" |

## 7. 验收标准

- [ ] CombinedView 左侧显示"组合策略分析"入口按钮，状态随已分箱变量数变化
- [ ] CrossBinningDialog 正确列出所有已分箱/未分箱变量，未分箱变量不可选
- [ ] 2变量交叉分析时展示正确的二维热力图（颜色、数值、悬停提示）
- [ ] 筛选参数（样本占比、坏账率倍数、Lift）正确生效
- [ ] 规则列表按坏账率降序排列，风险等级标注正确
- [ ] 导出功能生成正确的 Excel / 文本 / JSON 格式
- [ ] 10万样本 × 3变量 × 各5箱，分析耗时 < 5秒
- [ ] 组合数 > 5,000 时正确拒绝并提示
- [ ] 各变量的有效过滤规则在组合分析中被正确应用
- [ ] 代码遵循现有 MVC + Signal/Slot 架构风格

## 8. 后续扩展方向

1. **三维+可视化**: 支持 3 个变量的交互式 3D 柱状图展示
2. **联合最优分箱**: 集成 OptBinning2D 做真正的二维联合最优分箱（非笛卡尔积）
3. **规则冲突检测**: 自动检测导出规则之间的重叠和矛盾
4. **策略模拟**: 对选中的规则组合进行策略效果模拟（通过率、坏账率、盈利预测）
5. **规则版本管理**: 保存不同版本的策略规则集，支持对比和回滚
